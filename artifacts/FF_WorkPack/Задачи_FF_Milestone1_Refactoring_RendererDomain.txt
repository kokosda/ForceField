FF_Milestone1_Refactoring

=========================

Renderer Domain (связанный набор сущностей отображение)

Ожидаемый результат: получить связанную внутренне и удобную в использовании систему визуализации состояния сцены


Задачи:

1. Построение сущностей домена: Entity (междоменная), RenderEntity, Sprite. Должно быть организовано: Проект ForceFieldDomain, папка RendererDomain
2. Разработка класса Animator. Класс аниматор должен решать, какой "ключ" анимации нужно взять из последовательности в данный момент времени для данного спрайта. Он должен оперировать понятием Frame (отдельный класс, можно создать в домене, возможно потребуется создать в домене отдельный класс Animation при необходимости) анимации для разрешения этой задачи. Результат его работы отправляется в свойство RenderEntity->clientRect (отрисовываемая часть текстуры). Результатом реализации класса аниматор является замена всей текущей кинематики персонажей через вызов экземпляра данного класса над каждымм спрайтом (Sprite). На данном этапе может потребоваться создание вспомогательных классов для решения этой задачи, которых нету на текущей схеме FFDomain. Должно быть организовано: Проект ForceFieldCore, папка - корень проекта.
3. Разработка класса SpriteBatcher: необходимо реализовать возможность отрисовки множество объектов Sprite. Список объектов экземпляр класса получается из вне, возможно, в конструктор. Экземпляр класса создается при инициализации программы и помещается в событие Draw. Должно быть организовано: Проект ForceFieldCore, папка - корень проекта.

Стиль программирования в проекте:
а) названия пространств имен повторяют названия полных путей внутри проекта
б) организация синтаксиса классов
б.1) название классов с больших букв
б.2) члены классов в нижнем регистре без доп. символов, свойства с больших букв
б.3) в методах использовать глаголы; каждое слово в методе с большой буквы
б.4) "this" разрешается только в конструкторе; в любых других местах - только после согласования с тим-лидом
в) комментарии
в.1) оставлять по желанию
в.2) закомментированный код держать не более 2х коммитов на сервер репозитория (в коде не должно быть "зеленого мусора")
